# Architecture

The app is intentionally compact: a static `index.html`, one stylesheet, one script file, and local assets.

## Files

```text
index.html
styles.css
script.js
assets/
docs/
```

## Runtime Flow

1. `script.js` loads starter state or saved state from `localStorage`.
2. The editor controls are populated from state.
3. The preview is generated from state with `buildSiteHtml()`.
4. Layout-specific HTML is generated by `buildSectionHtml(section)`.
5. Export HTML is generated by `buildExportDocument()`.
6. Download uses the generated HTML and packages images into a ZIP.

## Main Concepts

- `state`: all editable site data
- `sections`: ordered page sections
- `groups`: navigation and parallax containers for sections
- `layouts`: rendering modes for each section
- `assets`: local image files used by the preview and export

## Persistence

State is saved in `localStorage` under:

```text
website-builder-state
```

If the `previewId` changes, the app resets to starter state.

## Rendering

The preview is not a separate iframe. It is generated inside:

```html
<article class="site-preview" id="sitePreview"></article>
```

This keeps the prototype simple and lets the editor and preview share one runtime.
